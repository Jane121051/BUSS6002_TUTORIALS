Linear Algebra Review¶

Linear algebra is the area of maths that deals with vector spaces and linear mappings between those spaces. Vectors are just an ordered list of numbers or variables, so we can represent any data as a vector. Take for instance the table of housing data below:

<img src="img/datatable.png" width = 320>

Each row can be represented as a vector in the $ \mathbf a, \mathbf b, \mathbf c $ space spanned by the basis vectors corresponding to the variables Area, Bathrooms and Price:

$$ \mathbf a = [1, 0, 0] $$ $$ \mathbf b = [0, 1, 0] $$ $$ \mathbf c = [0, 0, 1] $$

The 1st row can then be represented as a 3-dimensional vector by taking a weighted sum of the basis vectors:

$$ \mathbf x_1 = 350 \mathbf a + 3 \mathbf b + 1060000 \mathbf c = [350, 3, 1060000] $$

Similarly, each column of data can be represented as a vector in our sample space:

$$ \mathbf A = \begin{bmatrix}350\\200\\540\\275\end{bmatrix} $$

There is no reason why we could not represent our table columns as row vectors and our table rows as column vectors, we only need to be consistent in our choices.

Matrices are like vectors with an additional dimension, and can be thought of as a collection of vectors. Conversely, we can also think of vectors as lower-dimensional matrices. We can represent the data in the table above as a $4\times3$ matrix:

$$ \mathbf X = \begin{bmatrix}350 & 3 & 1060000\\200 & 2 & 620000 \\540 & 3 & 1490000\\275 & 3 & 900000\end{bmatrix} $$

We say this matrix is rectangular since 4 3. Matrices with an equal number of rows and columns ($m \times m$ matrices), are referred to as square matrices. These matrices are especially useful in that they can be invertible. This will be useful when dealing with matrix equations, but for now it's enough to know that square matrices are somewhat nicer to deal with than non-square ones.

Matrices are also commonly used to represent image data. For example, the entries in the below matrix represent the greyscale value of each pixel in the image.

Regression¶

Label, Features and Training Examples¶

A label is the variable which we are trying to estimate. This could be a house price, weather temperature or even a categorical label like brand of car. Usually we use the letter $y$ to denote the label variable. It is also known as the dependant variable or target variable.

A feature is an input variable which we use to estimate a label. We can have one or many features for each label. We use the letter $x$ to denote features. Some examples of features could be the size of a house, the number of bedrooms, distance to a school etc. When we have many features for a label we can write them as a vector $$ \mathbf x = [x_1, x_2, ..., x_p] $$

An example (or "training example") is a pairing of label and feature. We use training examples to train a model. In other words, given a set of data features $\mathbf x$ we adjust our model so that we get an accurate proediction for $y$ $$ [x_1, x_2, ..., x_p] \rightarrow y $$

An example of features and target for the house price example might include Square Feet, Bedrooms and Bathrooms as Features and the Price as the label.

<img src="img/feature_target_table.png" width = 500>

Training and Inference¶

Once we have determined what the target variable and feature variables are our next step is to build and test a model. There are two main stages to this process:

  • Training refers to the learning process. We give the model training examples and the model corrects itself to model or learn the relationships between the features and label.
  • Inference is the process of applying our model to new data. You use the trained model to make predictions.

<img src="img/training_inference.png" width = 800>

Linear Regression¶

A regression model means a model that predicts a label that is continuous based on a set of features. House prices and temperatures are examples of continuous variables that can be modeled using regression. Variables like brand of car or blood type are not continuous and can be desribed as discrete.

Generally there are two types of linear regression:

Simple Linear Regression $$ y = \beta_0 + \beta_1 x $$

Multiple Linear Regression $$ y = \beta_0 + \beta_1 x_1 + \dots + \beta_p x_2$$

A regression model can be uniquely described by the parameters $\beta$. They represent the effect or weight that each feature has on the value of the target. Training a regression model is the process of finding the best parameters to minimise the error between training target and predicted target values.

Simple means that we only use one feature in this process, whereas Multiple regression means that we use multiple features in the model to explain the target variable.

If we write the $\beta_i$ coefficients and features $x_i$ in vector form:

$$ \mathbf{\beta} = \begin{bmatrix}\beta_0\\\beta_1\\\beta_2\\...\end{bmatrix} \qquad \mathbf x = \begin{bmatrix}1&x_1&x_2&...\end{bmatrix} $$

We can write the equation as:

$$ \mathbf x \beta = y $$

If we expand the $\mathbf x$ vector into a matrix, with each row representing an observation in our dataset, we can write the set of linear equations

$$ \beta_0 + \beta_1 x_{11} + \beta_2 x_{12} ... \beta_p x_{1p}= y_1 $$ $$ \beta_0 + \beta_1 x_{21} + \beta_2 x_{22} ... \beta_p x_{2p}= y_2 $$ $$ \beta_0 + \beta_1 x_{31} + \beta_2 x_{32} ... \beta_p x_{3p}= y_3 $$ $$ ... $$ $$ \beta_0 + \beta_1 x_{n1} + \beta_2 x_{n2} ... \beta_p x_{np}= y_n $$

In compact matrix form:

$$\begin{bmatrix}1&x_{11}&x_{12}&...&x_{1p}\\1&x_{21}&x_{22}&...&x_{2p}\\ & &...\\1&x_{n1}&x_{n2}&...&x_{np}\end{bmatrix} \begin{bmatrix}\beta_0\\\beta_1\\\beta_2\\...\\\beta_p\end{bmatrix} = \begin{bmatrix}y_1\\y_2\\...\\y_n\end{bmatrix} $$

Ordinary Least Squares method¶

If we define the error as the difference between the target and the predicted value: $$ e = y - \beta_0 - \beta_1 x $$

then our objective becomes to minimise the square of the error: $$e^2 = ( y - \beta_0 - \beta_1 x ) ^ 2 $$

for all n training examples:

$$ \min_{\beta_0, \beta_1} \sum_{0}^{n}( y - \beta_0 - \beta_1 x_{i} ) ^ 2 $$

Framing the Problem¶

Given the Baton Rouge data we can do many regressions depending on which variable we select as the target or features. However there are two guiding questions you can ask yourself:

  • Which variable can you estimate from the others? (what is the dependency order of variables)
  • What is your goal?

In this example our goal is to estimate the house price. House price is the most obvious dependent variable. All others have only weak dependency on each other. For example we expect that a bigger house is more expensive. But a more expensive house is not always bigger (due to factors such as location).

Categorical variables¶

Notice that some variables are categorical. This presents a potential complication. Fortunately for us the variables have already been converted to "Dummy Variables". Dummy variables are also known as boolean indicator variables.

For each of the available categories a new variable is created. If the category is satisfied then that column is set to 1. The other columns are set to 0. Usually if there are $k$ categories then there will be $k-1$ dummy variables in a linear regression setting. This is to avoid multicolinearity.

  1. Scatter plotting of each independant variable against the dependant

The best way to do is to generate values for $x$ evenly spaced along the domain of the independant variable. Then for each value of $x$ calculate the value of the dependant variable.

Evaluating the Model¶

How can we quantify the ability of our model to reflect the data?

Goodness of fit¶

  • RMSE: the Root Mean Square Error measures the total error (distance between the regression line and actual dependant variable value for each training data point, summed over all training data points).
  • $R^2$: The R-squared value is a measure of the amount of variance that the model explains. A value of 1 means that the model totally explains all variance. However in most cases this would be considered overfitting.

Effect of variables¶

  • p-value: this is the statistical significance of the parameter value being non-zero. This does not mean that the variable should neccesarily be included in the model. You must also check the coefficient value. Often variables should be left in the regression even if their p-value is high so that you can indicate that you have controlled for this variable.
  • coefficient value: the magnitude of the coefficient/param value $\beta$. The larger this value the more it contributes to shifting the value of the dependant variable. A large coefficient value means the variable is practically significant. However since variables are at different scale you should standardise your variables first so their scales are the same. Some variables can't be compared in this way e.g. numeric vs categorical.

Explicit Linear Regression with Linear Algebra¶

We can also solve the linear regression problem directly using linear algebra. The goal is to minimise the squared error between the ground truth and the predictions from the linear model, defined by the coefficients in $\beta$. For a data matrix $X$

$$ X = \begin{bmatrix}1&x_{11}&x_{12}&...&x_{1p}\\1&x_{21}&x_{22}&...&x_{2p}\\ & &...\\1&x_{n1}&x_{n2}&...&x_{np}\end{bmatrix} $$

we can express the squared error in vector form:

$$ \sum_i\left(\sum_j X_{ij} \times \beta_j - y_i\right)^2 = || X\beta - \mathbf y ||^2$$

This minimisation problem has a unique solution as long as the columns of $X$ are linearly independent, which is similar to saying that our dataset does not have much multicolinearity.

Multiplying the expression by $ X^T$, we can recast the equation in terms of square matrices:

$$ X^T X \beta = X^T \mathbf y$$

Inverting the square matrix $ X^T X$, we can solve explicitly for $\beta$:

$$ \beta = \left( X^T X\right)^{-1} X^T \mathbf y$$

This expression can be evaluated directly in python by using .linalg.inv() to invert the square matrix.

We can also solve the equation using .linalg.solve(), which takes a matrix and a vector as input ($A$ and $\mathbf y$ for instance) and returns a vector $\mathbf x$ such that $A \mathbf x = \mathbf y$.

This is equivalent to solving the set of linear equations defined by $X^T X$.
